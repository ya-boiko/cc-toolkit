# git-commands

Git workflow commands for Claude Code — commit, PR description generation, GitLab MR creation, and working directory management.

## Install

```bash
./install.sh    # symlink all commands into ~/.claude/commands/
./update.sh     # git pull (symlinks update automatically)
./uninstall.sh  # remove symlinks
```

Adding a new `.md` file to `commands/` and re-running `./install.sh` makes it globally available.

## Workspace Mode

Commands adapt their behavior based on a `.workspace.md` file searched upward from `$PWD` to `$HOME`:

```markdown
---
mode: work
---
```

- **`work`** — enables team conventions: task numbers in commits, Jira links in PRs
- **`personal`** — minimal format, no task numbers required

If the file is not found, defaults to `personal`. Add `.workspace.md` to your global `.gitignore`.

## Commands

### `/commit`

Creates a git commit.

```
/commit [message] | --no-verify | --amend
```

**What it does:**

1. In `work` mode — asks for the Jira task number
2. Runs pre-commit checks by default: `pnpm lint`, `pnpm build`, `pnpm generate:docs`
3. Checks staged files — if nothing is staged, stages all changes automatically
4. Analyzes the diff to detect if changes should be split into multiple commits
5. Generates a commit message

**Commit format:**

- `work`: `[PRJ-1234] feat: description`
- `personal`: `feat: description`

**Commit types:** `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`

**Options:**

- `--no-verify` — skip pre-commit checks
- `--amend` — amend the last commit

### `/generate-pr`

Generates a Pull Request description from the current branch diff and commits.

```
/generate-pr [target-branch]
```

Default target branch: `master`.

**What it does:**

1. In `work` mode — asks for the Jira task number, Jira URL, and whether it's a continuation PR
2. Reads all commits between current branch and target
3. Analyzes the full diff
4. Generates a PR title and description, saves to `prs/<source>__to__<target>.md`

**Output format (work):**

```markdown
[PRJ-1234] Add Easter egg to the header

https://jira.example.com/browse/PRJ-1234

Adding the snowflake to the header:
- A clickable snowflake that brings up the modal
- The modal contains a mini-game with the cat
```

**Output format (personal):**

```markdown
Add Easter egg to the header

Adding the snowflake to the header:
- A clickable snowflake that brings up the modal
- The modal contains a mini-game with the cat
```

**Continuation PR title:** `[PRJ-1234.2] Fix snowflake animation`

**Example output path:** `prs/feat_PRJ-1234-easter-egg__to__master.md`

### `/mr`

Creates a GitLab Merge Request via `glab` using a description file generated by `/generate-pr`.

```
/mr
```

**Requirements:** `glab` installed and authenticated.

**What it does:**

1. Detects the current branch
2. Finds matching files in `prs/` (by branch name prefix)
3. If multiple files found — shows a numbered list with target branch and modification date, asks which to use
4. Parses the title (first line) and description (rest) from the chosen file
5. Generates a squash commit message from the MR title and content
6. Pushes the current branch: `git push -u origin <branch>`
7. Creates the MR with squash message: `glab mr create --title "..." --description "..." --squash-message "..." --target-branch <target>`
8. Returns the MR URL

**Squash commit message** — subject line + optional bullet body:
```
[PRJ-1234] feat: add Easter egg to the header

- clickable snowflake triggers a modal
- modal contains a mini-game with the cat
```

**Typical workflow:**

```
/generate-pr develop   → creates prs/feature-x__to__develop.md
/mr                    → pushes branch, creates MR from that file
```

### `/cd`

Switches the working directory for all subsequent operations in the current session.

```
/cd [directory]
```

Without arguments — lists subdirectories and asks which one to use. With an argument — switches immediately.

Useful when working from a parent directory that contains multiple projects (e.g. worktree directories created by `wt`).

**Recommended companion:** Add a SessionStart hook to `~/.claude/settings.json` that auto-detects multi-project directories and asks which one to use:

```json
{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "dirs=$(find . -maxdepth 1 -mindepth 1 -type d -exec test -e '{}/.git' ';' -print | sort | sed 's|^\\\\./||'); if [ -n \"$dirs\" ] && [ ! -e .git ]; then list=$(echo \"$dirs\" | paste -sd ', ' -); echo \"{\\\"hookSpecificOutput\\\":{\\\"hookEventName\\\":\\\"SessionStart\\\",\\\"additionalContext\\\":\\\"This directory contains multiple projects: $list. Ask the user which project directory to use as the working directory for this session. Once they choose, cd into it for all subsequent work.\\\"}}\"; fi"
          }
        ]
      }
    ]
  }
}
```

The hook fires only when the current directory has no `.git` but contains subdirectories with `.git`. Inside a single project — silent.
