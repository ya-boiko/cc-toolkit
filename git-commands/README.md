# git-commands

Git workflow commands for Claude Code — commit, PR description generation, and GitLab MR creation.

## Install

```bash
./install.sh    # symlink all commands into ~/.claude/commands/
./update.sh     # git pull (symlinks update automatically)
./uninstall.sh  # remove symlinks
```

Adding a new `.md` file to `commands/` and re-running `./install.sh` makes it globally available.

## Commands

### `/commit`

Creates a git commit in [conventional commits](https://www.conventionalcommits.org/) format.

```
/commit [message] | --no-verify | --amend
```

**What it does:**

1. Runs pre-commit checks by default: `pnpm lint`, `pnpm build`, `pnpm generate:docs`
2. Checks staged files — if nothing is staged, stages all changes automatically
3. Analyzes the diff to detect if changes should be split into multiple commits
4. Generates a commit message following conventional format: `<type>: <description>`

**Commit types:** `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`

**Options:**

- `--no-verify` — skip pre-commit checks
- `--amend` — amend the last commit

---

### `/generate-pr`

Generates a Pull Request description from the current branch diff and commits.

```
/generate-pr [target-branch]
```

Default target branch: `master`.

**What it does:**

1. Reads all commits between current branch and target
2. Analyzes the full diff
3. Generates a PR title (max 72 characters) and structured description
4. Saves the output to `prs/<source>__to__<target>.md`

**Output file format:**

```markdown
<PR Title>

## Summary

<Why these changes were made>

## Changes

### <Category>
- Change 1
- Change 2
```

**Example output path:** `prs/PROJ-123_add-caching__to__develop.md`

---

### `/mr`

Creates a GitLab Merge Request via `glab` using a description file generated by `/generate-pr`.

```
/mr
```

**Requirements:** `glab` installed and authenticated.

**What it does:**

1. Detects the current branch
2. Finds matching files in `prs/` (by branch name prefix)
3. If multiple files found — shows a numbered list with target branch and modification date, asks which to use
4. Parses the title (first line) and description (rest) from the chosen file
5. Pushes the current branch: `git push -u origin <branch>`
6. Creates the MR: `glab mr create --title "..." --description "..." --target-branch <target>`
7. Returns the MR URL

**Typical workflow:**

```
/generate-pr develop   → creates prs/feature-x__to__develop.md
/mr                    → pushes branch, creates MR from that file
```
